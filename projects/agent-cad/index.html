About

Agent CAD is a full CAD/CAM system built from scratch so that AI agents can author geometry directly. The core insight: SDF (Signed Distance Functions) as the modeling kernel means booleans never fail — the #1 pain point in traditional B-Rep CAD. An LLM says what it wants, and the kernel resolves the geometry.

The system exposes 44 MCP tools that let any LLM create, modify, and manufacture parts. Six primitives, boolean operations (with smooth variants), transforms, modifiers, and a semantic DSL with machining vocabulary: hole(), pocket(), boltCircle(), chamfer(), fillet().

Architecture

Monorepo with three packages:

  • sdf-kernel — TypeScript SDF kernel with 20 node classes, named topology for analytical face/edge identity
  • mcp-server — MCP server exposing the full kernel API to LLMs
  • viewer — Vite + Three.js + CodeMirror 6 viewer with face highlighting, dimension readout, dark charcoal aesthetic

CAM Pipeline

Agent CAD isn't just modeling — it includes a complete CAM pipeline: 3-axis raster surfacing, 2D contour/profile, multi-level contour roughing, and Fanuc G-code output. The vision is end-to-end: an agent designs a part and generates the code to cut it.

The Vision

Hybrid kernel architecture: Fidget (SDF/WASM) for modeling, OCCT (B-Rep/WASM) at the export boundary, Manifold for mesh booleans. The kernel is self-describing — structured JSON readback after every operation gives the LLM full context of what it just built. Intent-based API means the agent works at the level of machining intent, not raw geometry commands.
